# Security Policy

## Supported versions

| Version | Supported |
|---|---|
| 0.1.x | Yes |

## Reporting a vulnerability

Use GitHub's private vulnerability reporting or a Security Advisory.

Do not open a public issue containing:

- credentials or access tokens;
- private repository content;
- personal data;
- production paths or logs;
- exploit details that create immediate risk.

## Scope

Security reports may cover:

- secret-detection bypasses;
- unsafe file handling;
- path traversal;
- symlink handling;
- denial-of-service conditions;
- report injection;
- configuration bypasses;
- unsafe CI behavior.

## Disclosure

The maintainer will acknowledge valid reports, assess impact, and coordinate a
fix and release. Response times are best effort; this is a small OSS project.
